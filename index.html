THE STARTING POINT

A Fragmented Ecosystem

When I joined Forvia, the data ecosystem was powerful but deeply fragmented. Different products were built at different times, by different teams, with different design decisions.

  • Interfaces that felt unrelated to each other
  • Components behaving inconsistently
  • Different interaction patterns across tools
  • Visual identity that shifted from app to app
  • Users needing to relearn UI logic constantly
OBJECTIVE

Transform fragmented tools into a unified ecosystem

Rather than shipping a single feature or redesigning just one product, my work focused on orchestrating a cohesive, scalable user experience across the entire data platform.

1. Dashboards

Designing intuitive, insight-driven interfaces that enable faster decision-making

2. Product Ecosystem

Aligning multiple tools under a consistent visual and behavioral language

3. Design System

Building a scalable foundation to sustain consistency as the platform evolves

TIMELINE OF IMPACT

Five Strategic Phases

1

Understanding & UX Audits

Mapped the ecosystem, audited every tool, identified patterns and user frustrations

2

Designing Better Dashboards

Improved clarity, readability, and structure for faster insights

3

Unifying the Ecosystem

Introduced consistent patterns, behaviors, and visual logic

4

Building The Design System

Created Palantir-compatible system with components, tokens, and documentation

5

Alignment & Collaboration

Facilitated workshops to align teams and guide strategic decisions

IMPACT

Measurable Business Results

  • Unified experience across 8+ disconnected data tools
  • Reduced onboarding time for new users across the platform
  • Improved decision-making speed through clearer dashboard design
  • Decreased design and development friction via shared system
  • Scalable foundation enabling consistent future product development
  • Strengthened cross-team collaboration through Paris workshop alignment

This wasn't cosmetic work—it was experience architecture that transformed how teams interact with enterprise data.
